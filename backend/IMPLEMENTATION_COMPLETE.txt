For real email, update `.env`:
```env

  -d '{"email": "test@example.com"}'

# Check console for OTP code
# Then verify and register
```

## API Examples

### Step 1: Send OTP
```bash
POST /api/auth/send-otp/
{
  "email": "user@example.com"
}

Response:
{
  "message": "OTP sent successfully to your email",
  "email": "user@example.com",
  "expires_in": "10 minutes"
}
```

### Step 2: Verify OTP
```bash
POST /api/auth/verify-otp/
{
  "email": "user@example.com",
  "otp_code": "123456"
}

Response:
{
  "message": "Email verified successfully.",
  "verified": true
}
```

### Step 3: Register
```bash
POST /api/auth/register-with-otp/
{
  "username": "johndoe",
  "email": "user@example.com",
  "password": "SecurePass123!",
  "password_confirm": "SecurePass123!",
  "otp_code": "123456"
}

Response:
{
  "user": {...},
  "token": "eyJ0eXAi...",
  "message": "Registration successful!"
}
```

## Troubleshooting

### Emails not appearing?
Check `.env` EMAIL_BACKEND setting
- Console: Should print to terminal
- SMTP: Check credentials and gmail app password

### OTP expired?
Use `/auth/resend-otp/` to send new code

### Database error?
Run: `python manage.py migrate`

### Import error?
Run: `python manage.py check`

### Can't register?
Make sure OTP is verified first
